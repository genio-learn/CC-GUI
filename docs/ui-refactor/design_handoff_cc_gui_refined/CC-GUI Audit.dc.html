Design Audit · CC-GUI

CC-GUI: flows, screens & a refined redesign

A full audit of claude-commander's desktop GUI — every core flow and screen mapped, then critiqued for discoverability, visual consistency, density, and modern polish. Each screen is shown as it is today, then re-mocked in the Refined direction: the same Catppuccin-Mocha DNA, sharpened — labeled states, inline actions, roomier scale, nothing clipped.

Pan & zoom freely. Grounded in the live source — index.html, style.css tokens, and the design handoff in docs/ui-overhaul.

01

The core flows & screens

CC-GUI is mission control for parallel Claude Code agents. Everything hangs off six flows. Each is a screen (or overlay) audited and redesigned below.

▤

1 · Triage & attach

Scan every agent session by state, pick one, attach a live terminal. The Console sidebar + the Board.

❯

2 · Work a session

Live PTY terminal, tabs, split panes, the detail pane, project shells, the file explorer (⌘E).

±

3 · Review & comment

Read the diff, stage inline comments, Apply them back to the agent.

⌗

4 · Cascade & lifecycle

Stacks: merge / push / resume. Per-session: rename, restart, restart fresh, kill vs delete.

⌘

5 · Navigate & command

The ⌘K palette (jump + run), the ? help overlay, keyboard-first movement.

◐

6 · Configure

Projects, sections, theming (10 built-ins + custom), settings & telemetry.

Add a project · new session (+) · project shell ($) · commander session are the connective tissue between them.

02

What the audit found

Fourteen issues across the four lenses you weighted: discoverability, visual consistency, density & scannability, and modern polish. Ordered by severity. Every fix is applied in the mockups that follow.

HighDiscoverability

Primary actions live far from where you click

You select a session in the far-left sidebar, but "± Review diff" and "Open PR" sit in the bottom-right of the detail pane — a full diagonal mouse trip. Attach is the only thing that happens where you click.

Fix: surface Review / Shell / PR as inline row actions that appear on the session you're pointing at, and pin a compact action bar to the top of the detail pane.
HighConsistency

Two warnings, nearly the same color

Blocked-auto-pull uses maroon ⚠; a separate "pull blocked" uses peach ⚠. Same glyph, adjacent hues — indistinguishable, and neither is labeled.

Fix: one warning treatment, a labeled pill ("Pull blocked"), reserved color, never two ambiguous shades of the same idea.
HighDiscoverability

The glyph vocabulary is a private language

✎ ⚠ ⌗ ⚡ ◷ ⛙ ↑ $ ◐ ⇄ — a dozen Unicode symbols carry critical meaning with no labels. $ = project shell, ⌗ = stack, ⛙ = merge. Unguessable; only the ? overlay explains them.

Fix: pair every glyph with a text label (or persistent tooltip), shrink the vocabulary, and adopt one consistent icon treatment.
HighDensity

Type & targets run small

Core labels at 10–11px, session names 13px, icon buttons 21px, session rows ~30px tall. On a big display it's cramped and the hit targets are fiddly — the pain you flagged.

Fix: lift the base scale, floor interactive targets at ~28–32px, and widen row padding for a calmer rhythm.
MediumConsistency

Status is carried by color alone

Running / finished / idle / stopped are all 8px dots that differ only in hue. Colorblind users can't separate green from yellow from sky, and a dot never says what it means.

Fix: encode state with shape + color + label — a labeled status chip, not a bare dot.
MediumConsistency

Four different segmented controls

The title-bar toggle, GROUP BY, the settings mode switch, and the Board filter pills are each styled differently — four takes on "pick one of a set."

Fix: one segmented-control primitive and one pill primitive, reused everywhere.
MediumConsistency

Button & radius sprawl

row-action, proj-btn, board-pill, card-action, titlebar buttons all differ in size, border and radius. Corner radii span 4/5/6/7/8/10/11/14px with no scale.

Fix: a single button system (3 sizes, 3 intents) and a 4-step radius scale.
MediumPolish

Board cards clip their own data

In the shipped Board, the diffstat counts (−44) get cut off at the column edge, and cards overflow the fold. Content is being truncated, not laid out.

Fix: give cards a real content budget, wrap/relayout diffstat, and make the accent state legible at a glance.
MediumDiscoverability

No first-run / empty state

With zero projects there's nothing to guide a new user in — the sidebar is just empty. The whole "add a project → start a session" onramp is invisible.

Fix: a proper first-run screen (new flow) that teaches add-project, new-session, and the commander.
MediumDensity

Kill vs Delete are a trap

In the context menu, "Kill — stop process" (yellow) sits one line above "Delete session…" (red). Similar rows, adjacent, both destructive-looking — easy to misfire under a fast right-click.

Fix: separate them visually, label the consequence on each, and keep the typed/confirmed guard on Delete.
LowPolish

Depth & motion are flat

Only the running dot animates. Panels sit on one elevation plane; state changes, attach, and tab switches happen with no transition, so the eye loses the thing that moved.

Fix: a small elevation scale and short, purposeful transitions on state, selection, and overlay entry.
LowDiscoverability

"Needs review" is a section, not a signal

The one thing you most want — what's waiting on me — is buried as a list group. There's no at-a-glance count of the queue across projects.

Fix: promote an attention summary to the title bar and the top of the Board.
LowDensity

Detail pane repeats what the row shows

Branch, status, and diffstat appear in the row, the powerline, and the detail pane — three times, three formats. Redundant without being reinforcing.

Fix: let each surface own one job; the detail pane goes deeper (summary, files, PR) rather than restating.
LowPolish

Mono font drift in the chrome

Chrome labels mix MesloLGS, Menlo, and ui-monospace. Small, but it reads as three slightly different "code" voices in one window.

Fix: one UI sans, one code mono, applied by role — terminal keeps MesloLGS for glyphs.

09

The call — ship Refined

One direction, chosen. Refined fixes every audit item without a rewrite — same tokens, same layout skeleton, the 10 themes reskin untouched.

Refined

Low-risk, mergeable screen-by-screen

The wins — labeled states, inline actions, roomier scale, unclipped Board cards, a review progress ring and a sticky Apply bar — are mostly CSS + small DOM moves against the existing imperative render. No new navigation model, no framework change. It reads as "the app, finally tuned."

Sequence: Console first (highest traffic), then Board, then Review.
New flows

Land the two things that were missing

Two fixes are net-new rather than restyles: first-run onboarding and the decoded status vocabulary. Build them alongside the Console pass so a new user is onboarded straight into the labeled-state system from launch.

Effort: one empty-state screen + a shared status-chip reused on every surface.

Recap of what's fixed: actions moved to point-of-use · glyphs labeled · one warning color · shape+color+word status · larger type & targets · unified buttons/segments/radii · unclipped Board cards · Kill≠Delete guard · first-run onboarding · attention surfaced up top.
